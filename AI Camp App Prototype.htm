Build Plan: "Orbs" (HobbySphere)

What I'm Building

A single-page React app (Vite) that feels like a native mobile app — a personal sanctuary for hobbies. Four screens: Home (orb wall), Interest Detail (album + journal tabs), Add/Edit modals, and Profile. All data lives in IndexedDB via Dexie.js. No backend, no login, works offline.

Architecture Decisions

Tech Stack
  • React 18 + Vite (fast dev, clean build)
  • React Router (4 routes: /, /interest/:id, /profile)
  • Dexie.js for IndexedDB (stores interests, photos, entries with the exact schema from the PRD)
  • Tailwind CSS for styling (soft, warm, rounded aesthetic — no sharp corners)
  • No external UI library; custom components for the "glowing orb" feel
Data Model (exactly per PRD Section 7)
  • interests: id, name, color (gradient key), why, createdAt, updatedAt
  • photos: id, interestId, blob, caption, isPinned, createdAt
  • entries: id, interestId, date, text, isPinned, createdAt, updatedAt
Derived data computed on-the-fly: orb face photo (newest photo per interest), streaks (consecutive days with entries), stats.
Storage Strategy
  • IndexedDB stores everything. Images stored as Blobs, rendered via URL.createObjectURL() with cleanup on unmount.
  • Images downscaled to ~1600px max edge before storage to prevent quota issues.
  • No localStorage (5MB cap, can't store blobs).
  • Known risk: browser can clear data. Accepted for v1; export/import as stretch.
PWA: Web manifest + 512x512 icon so it can be "Add to Home Screen" and open fullscreen.

Key Design Choices

Aesthetic: Soft, warm, glowing. Rounded everything. Gradient orb cards with subtle float animation. No harsh shadows — diffuse glow only. Color palette: 8–10 curated gradients (warm pinks, soft blues, gentle greens, sunset oranges).
Memory Banner: Shows one pinned item (photo or entry) at random, or falls back to most recent. Warm prompt text: "Remember this?" This is the emotional hook.
"Why I love this": Editable on each interest. Gives each orb personality.
Streaks: Computed from entry dates (consecutive days with at least one entry per interest). Displayed subtly — no pressure, no "streak freeze" (per Constitution).
No Gamification: No badges, no points, no leaderboards, no public sharing. The reward is seeing your own gallery grow (per Constitution).
Gaming Exclusion: If a user names an interest "Gaming" or similar, the app allows it but excludes hours/entries from streak calculations and totals. Gentle signal, not a ban.
Localization: UI in English for this build, but all text extracted to a config object so Chinese can be swapped in easily.

Risk Mitigation

  • Browser storage cleared: Accepted risk. Export/import JSON stretch if time allows.
  • Image quota exceeded: Downscale to 1600px max edge. Warn if storage >80% full.
  • Mobile file picker: Use <input type="file" accept="image/*" capture> for camera + gallery.
  • Object URL leaks: Revoke all URL.createObjectURL() in cleanup effects.
